# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

AdsVizor is a multi-tenant landing page + lead capture system. Each client gets a subdomain (e.g. `formations.adsvizor.com`) served by **Cloudflare Pages**, which auto-deploys on every `git push` to `main`. There is no build step — everything is vanilla HTML/CSS/JS.

A separate **Cloudflare Worker** (`cloudflare-worker.js`, deployed as `adsvizor-leads`) acts as a CORS proxy, forwarding lead POST requests from the browser to a Google Apps Script endpoint, which writes to Google Sheets.

## Deployment

- `git push` to `main` → Cloudflare Pages deploys static assets automatically
- Worker deployed via `npx wrangler deploy` — targets worker named `adsvizor-leads` (configured in `wrangler.jsonc`)
- Formation static pages: `node scripts/generate-formation-pages.js` → regenerates `clients/{slug}/pages/{category}/formation-*.html` + `sitemap.xml`
- Apps Script: copy-paste `apps-script/Code.gs` → Google Apps Script editor → Deploy new version

## Local development

Use `wrangler pages dev` (NOT Live Server) — required to run `functions/_middleware.js` locally:

```bash
npm run dev          # starts wrangler pages dev on port 5500
# or directly:
npx wrangler pages dev . --port 5500 --compatibility-date 2026-04-19
```

Then open `http://localhost:5500/?client=formations` for the landing page.
Client-specific pages work automatically: `http://localhost:5500/formations.html?client=formations`

> **Why not Live Server?** Client-specific pages are stored under `clients/{slug}/pages/` and `clients/{slug}/blog/`. The middleware routes them at runtime. Live Server has no middleware support.

## How the template system works

`index.html`, `blog.html`, `contact.html` are shared templates using `{{placeholder}}` syntax. At runtime, `script.js`:

1. Detects the client slug from the subdomain, or falls back to `?client=` URL param
2. Fetches `clients/{client_slug}/config.json`
3. Walks the DOM and replaces all `{{placeholders}}` in text nodes and attributes
4. Sets `document.body.classList.add("ready")` to reveal the page (FOUC prevention)

**Static formation pages** (`clients/{slug}/pages/formation-{slug}.html`) are generated by `scripts/generate-formation-pages.js` — they have hardcoded SEO content (title, meta, H1) for Google Ads Quality Score. Run the generator after changing formation content in `config.json`.

## File responsibilities

- `index.html` / `blog.html` / `contact.html` / `thank-you.html` / `privacy.html` — shared templates
- `landing.html` — served on `adsvizor.com` root domain (logo only, no client content)
- `main.css` — all styles; mobile-first, breakpoint at 768px
- `script.js` — config loading, placeholder rendering, UTM capture, multi-step form, analytics
- `cloudflare-worker.js` — CORS proxy for `/api/leads`; injects consent_ip, consent_user_agent, visitor_city, visitor_region from Cloudflare headers
- `wrangler.jsonc` — wrangler config, targets worker `adsvizor-leads`
- `_redirects` — Cloudflare Pages 301 redirects (formations-cpf → formations; old flat URLs → new category URLs)
- `sitemap.xml` — auto-generated by `generate-formation-pages.js`
- `apps-script/Code.gs` — Google Apps Script source (v6, 26 columns A–Z)
- `clients/{slug}/config.json` — all client-specific text and settings
- `clients/{slug}/agent.config.json` — blog agent prompts and nav links
- `clients/{slug}/pages/formations.html` — formations listing page
- `clients/{slug}/pages/{category}/formation-*.html` — static formation pages organised by category (bureautique, langues, management, marketing, finance, dev-personnel, entrepreneuriat, ia)
- `scripts/generate-formation-pages.js` — generates static pages + sitemap from config.json
- `functions/_middleware.js` — routes *.html to client directory; serves landing.html on root domain; handles 301 redirects via REDIRECTS_301 map (old flat URLs → new category URLs)

## Key config.json fields

- `client_slug`, `offer_id`, `page_version` — included in lead payloads
- `form_action` — the Apps Script endpoint URL
- `nav_item_0_href` / `nav_item_0_label` — Accueil (home) link
- `nav_item_1_href` through `nav_item_4_href` — nav items
- Blog keys: `post_1_*` through `post_10_*`
- Why-us section: `why_us_title`, `why_us_a_*` through `why_us_d_*`
- Formation dropdown: `field_formation_label`, `field_formation_opt_*`, `field_formation_permis_msg`
- Contact process: `contact_step_1_*` through `contact_step_5_*` (with `_image`, `_icon`, `_title`, `_text`)

## Lead submission flow

Browser → POST JSON to Cloudflare Worker (`adsvizor-leads`) → Worker injects server-side fields → forwarded to Google Apps Script → appended to Google Sheet (26 columns A–Z).

**Worker injects (server-side, authoritative):**
- `consent_ip` — from `CF-Connecting-IP` header
- `consent_user_agent` — from `User-Agent` header
- `visitor_city`, `visitor_region` — from `request.cf` geolocation

**Frontend sends:**
- `consent_url`, `consent_text`, `consent_timestamp` — RGPD proof
- `formation_interest` — from dropdown or hidden input
- Standard UTM fields, professional_status, etc.

Multi-step form: Step 1 (name/phone/email/consent) → partial send on "Continuer" → Step 2 (status/formation) → full send on submit → redirect to `thank-you.html?code=XXXXXX`.

## Formation pages structure (2026-04-30)

Pages are organised under `clients/formations/pages/` by category:

```
pages/
├── formations.html              ← listing page
├── bureautique/                 ← Excel, Word, PowerPoint, Outils Collaboratifs, WordPress, PAO, Bases Informatiques
├── langues/                     ← Anglais TOEIC
├── management/                  ← Management Leadership, Gestion de Projet
├── marketing/                   ← Marketing Digital
├── finance/                     ← Comptabilité & Paie
├── dev-personnel/               ← Développement Personnel, Bilan de Compétences
├── entrepreneuriat/             ← Création d'Entreprise
└── ia/                          ← Intelligence Artificielle
```

Old flat URLs (`/formation-excel.html`) are handled by the `REDIRECTS_301` map in `_middleware.js` which returns 301 → new URL before any asset lookup. The `_redirects` file also has these rules as a safety net.

**IMPORTANT:** When adding a new formation page, also add a redirect entry in `REDIRECTS_301` in `functions/_middleware.js` if you ever need to change its URL later.

## Google Ads setup

- Formation pages serve as landing pages: `formations.adsvizor.com/{category}/formation-{slug}.html`
- Tracking template: `{lpurl}?utm_source=google&utm_medium=cpc&utm_campaign={campaign}&utm_term={keyword}&utm_content={adgroupid}`
- Conversion tracking: `gtag('event', 'conversion')` fires on `thank-you.html` when `?code=` param present (production only)
- Google tag IDs: `GT-KD7C7TR3` (Google Tag), `AW-18122720723` (Google Ads)

## Analytics events

Emitted via `console.log("[adsvizor_event]", ...)`:
- `page_view` — on every page load
- `cta_click` — on `[data-cta-id]` click
- `form_start` — on first focus/input
- `form_submit` — on submit attempt
- `form_partial` — on step 1 completion or permis disqualification

## CORS / Worker origin allowlist

`cloudflare-worker.js` allows: `adsvizor.com`, `www.adsvizor.com`, `*.adsvizor.com`, `localhost:*`, `127.0.0.1:*`.

## Conventions

- Client slugs: lowercase `a-z0-9` and `-` only
- No frameworks, no build steps, no TypeScript
- JS: `const` by default, `async/await` for fetch, no implicit globals
- After changing formation content: re-run `generate-formation-pages.js` and commit
- Worker deploy: `npx wrangler deploy` (not `versions upload/deploy`)
- Apps Script deploy: paste `apps-script/Code.gs` → new version in Google Apps Script console
