Blog Agent — Architecture

Pipeline de génération automatique d'articles · AdsVizor

V1
Pipeline fonctionnel
Objectif : publier le premier article automatique
⏰
1. Déclencheur — GitHub Actions Cron
Workflow YAML qui tourne automatiquement toutes les 2 semaines, sans intervention humaine. Peut aussi être déclenché manuellement via workflow_dispatch.
  • Schedule : 0 8 1,15 * * (1er et 15 du mois, 8h)
  • Environnement : Ubuntu, Node.js 20
GitHub Actions
🎯
2. Choix du sujet
Rotation automatique entre 3 types d'articles pour varier le contenu :
  • Témoignage — histoire de reconversion CPF
  • Actualités — réformes, nouvelles règles CPF
  • Formation — spotlight sur une formation tendance
Le type suivant est déterminé en lisant data/blog-history.json.
Node.js
✍️
3. Génération de l'article — Prompt 1
Appel Claude API avec un prompt structuré qui impose le template HTML exact (copié depuis blog-sophie-marchand.html). Claude génère le contenu dans la structure.
  • Modèle : claude-opus-4 ou claude-sonnet-4-5
  • Output : HTML complet + titre + excerpt pour config.json
  • Contraintes : date du jour, liens internes, style cohérent
Claude API
⚠️ Gestion d'erreur — à chaque étape Si Claude renvoie une erreur ou un contenu vide → process.exit(1) → le workflow échoue → GitHub crée automatiquement une Issue "Blog agent failed" avec le log d'erreur.
🔍
4. Optimisation SEO — Prompt 2
Second appel Claude : relit l'article généré et optimise sans réécrire.
  • Mot-clé principal dans le title, H1 et premier paragraphe
  • Meta description ≤ 155 caractères
  • Balises alt sur les éventuelles images
  • Structure des H2 logique pour les crawlers
Claude API
📣
5. Injection CTA automatique
Le script injecte le bloc CTA standard en fin d'article — sans passer par Claude, donc toujours identique et jamais oublié.
  • Bloc HTML hardcodé (copié depuis les articles existants)
  • Lien toujours vers contact.html
  • Texte du CTA varié selon le type d'article
Node.js
✅
6. Vérification avant commit
Parse le HTML avec cheerio et vérifie :
  • Aucun {{placeholder}} non résolu
  • Un seul H1, au moins 2 H2
  • Longueur ≥ 800 mots
  • CTA présent (.article-cta-block)
  • Liens internes valides (contact.html, blog.html)
  • Mot-clé principal dans le H1
Échec sur n'importe quel point → arrêt immédiat.
cheerio Critique
📚
7. Mise à jour historique
Écrit dans data/blog-history.json : titre, slug, type, date, mot-clé. Commité avec l'article pour persister entre les runs.
  • Évite les doublons de sujets
  • Détermine le prochain type d'article (rotation)
  • Met à jour config.json (post_1, post_2, post_3)
Node.js
🚀
8. Commit + Push → Déploiement auto
git add + git commit + git push origin main → Cloudflare Pages détecte le push et redéploie automatiquement en ~60 secondes.
git Cloudflare Pages
📧
9. Notification email
Email automatique via GitHub Actions (ou SendGrid) avec : titre de l'article, URL, mot-clé ciblé, type de contenu. Pour savoir ce qui a été publié sans aller sur le site.
GitHub Actions
V2
Pipeline production-ready
Objectif : zéro erreur factuelle, SEO maximal
⏰
1. Déclencheur — identique V1
Même cron GitHub Actions. La différence : le workflow pousse sur blog-drafts au lieu de main.
GitHub Actions
🔒
2. Idempotence — protection double-run Nouveau
Avant de générer quoi que ce soit, calcule un hash du sujet+date et vérifie dans blog-history.json.
  • Si le hash existe déjà → arrêt propre (pas d'erreur, juste skip)
  • Protège contre les crons qui se déclenchent 2 fois
  • Protège contre les relances manuelles accidentelles
Node.js
🎯
3. Choix du sujet — identique V1
Rotation témoignage / actualités / formation basée sur l'historique.
Node.js
🔄 Fallback si Claude échoue Retry automatique 1 fois avec un prompt simplifié. Si le second appel échoue aussi → process.exit(1) + GitHub Issue. Le sujet est conservé dans une file pour le prochain run.
✍️
4. Génération + Template strict Amélioré
Même appel Claude, mais avec un template XML imposé dans le prompt system. Claude doit remplir des balises prédéfinies — pas de HTML libre.
  • Zéro dérive de structure dans le temps
  • Output parsé via XML avant injection dans le template HTML
  • Garantit que le style reste cohérent après 20 articles
Claude API
🔍
5. Optimisation SEO — identique V1
Second prompt Claude dédié SEO.
Claude API
🔗
6. Internal linking automatique Nouveau
Le script lit la liste des articles existants dans blog-history.json et injecte 1 à 2 liens internes contextuels dans le corps de l'article.
  • Ex : un article sur l'IA → lien vers l'article Sophie (reconversion)
  • Ex : un article CPF → lien vers la page formations-cpf.html
  • Impact SEO majeur : réduit le taux de rebond, améliore le maillage
Node.js
📣
7. Injection CTA — identique V1
Bloc CTA hardcodé injecté en fin d'article.
Node.js
✅
8. Vérification — identique V1
Cheerio + checks structure, mots, placeholders, CTA, liens.
cheerio
🧠
9. Auto-review factuelle — Prompt 3 Nouveau
Troisième appel Claude avec un rôle de fact-checker CPF. Lit l'article et vérifie :
  • Montants CPF corrects (500 €/an, plafond 5 000 €, reste à charge 150 €…)
  • Références légales exactes (RNCP, Qualiopi, France Compétences)
  • Aucune promesse commerciale excessive ou non vérifiable
  • Clarté et crédibilité du discours
Output : PASS ou FAIL + raisons. En cas de FAIL → arrêt + GitHub Issue.
Claude API Critique CPF
🚦
10. Smart Gate — branch blog-drafts Nouveau
L'article est poussé sur blog-drafts, pas sur main. Cloudflare génère une URL de preview automatiquement.
  • Preview live accessible avant publication
  • Auto-merge vers main après 24h si aucun blocage
  • Ou merge manuel si tu veux relire
GitHub Cloudflare Preview
📚
11. Mise à jour historique — identique V1
data/blog-history.json mis à jour et commité. Inclut maintenant le score de fact-check et le hash d'idempotence.
Node.js
🚀
12. Déploiement — via merge vers main
Après auto-merge (ou merge manuel) de blog-drafts → main, Cloudflare Pages déploie en production.
Cloudflare Pages
📧
13. Notification — email + URL preview Amélioré
Email avec : titre, URL preview Cloudflare, URL production, mot-clé, score fact-check, lien pour merger manuellement si besoin.
