AdsVizor Web — Service Architecture

Generated 2026-04-19  |  adsvizor.com

1. Porkbun LIVE

Role: Domain registrar — owns and manages adsvizor.com.

  • Delegates DNS authority to Cloudflare nameservers
  • Email routing: contact@adsvizor.com → adsvizor@gmail.com
  • No code artifacts — purely registrar/DNS level configuration
Porkbun is the entry point of the entire domain. Everything else depends on it pointing to Cloudflare.

2. Cloudflare LIVE

Role: CDN, static hosting, serverless proxy, DNS, and SSL for the entire platform.

Cloudflare operates in three distinct layers:

LayerServiceWhat it does
Static hosting Cloudflare Pages Auto-deploys the site on every git push to GitHub. Serves HTML, CSS, JS, and client config files.
API proxy Cloudflare Workers Acts as a CORS bridge between the visitor's browser and Google Apps Script. Configured in wrangler.jsonc / cloudflare-worker.js.
DNS + SSL Cloudflare DNS Routes adsvizor.com and all *.adsvizor.com subdomains. Provides free HTTPS and DDoS protection.

Worker — CORS Allowed Origins

  • https://adsvizor.com
  • https://www.adsvizor.com
  • https://*.adsvizor.com (any client subdomain)
  • http://localhost:5500 / http://127.0.0.1:5500 (local dev)

3. GitHub LIVE

Role: Source control and deployment trigger.

  • Repository: adsvizor/adsvizor-web
  • A git push to main automatically triggers a Cloudflare Pages deployment
  • No separate CI/CD pipeline — Cloudflare Pages handles it natively

4. Google Services LIVE

4.1 Google Apps Script (Backend)

Serverless backend that receives lead submissions forwarded by the Cloudflare Worker.

  • Validates and normalizes the incoming JSON payload
  • Appends a row to Google Sheets
  • Returns a JSON success/failure response

Lead payload schema (POST body)

{
  "client_slug":        "formations",
  "offer_id":           "lead-gen-v1",
  "visitor_name":       "string | null",
  "visitor_email":      "string",
  "visitor_phone":      "string | null",
  "visitor_message":    "string | null",
  "utm": {
    "source":   "string | null",
    "medium":   "string | null",
    "campaign": "string | null",
    "term":     "string | null",
    "content":  "string | null"
  },
  "page_version":        "1.0.0",
  "consent_marketing":   true | false | null
}

4.3 Google Fonts

Delivers the Inter typeface (weights 400/500/600/700) via Google's CDN to all landing pages.

5. Claude API (Anthropic) PLANNED

Role: AI optimization agent that runs daily to improve campaign performance.

  • Reads leads from Google Sheets + metrics from Google Ads API
  • Generates ranked recommendations across 5 categories: bidding, targeting, keywords, ads, landing copy
  • Performs compliance QA on generated text
  • Outputs structured JSON stored for human review (or auto-apply)
Prompt engineering documented in docs/PROMPTS.md. Not yet deployed.

8. Complete Data Flow

VISITOR JOURNEY
───────────────
Visitor → Cloudflare Pages (HTML / CSS / JS / client config)
        → Google Fonts (Inter typeface)
        → Form submit → Cloudflare Worker /api/leads (POST)
                             → Google Apps Script
                                   → Google Sheets (row append)

DEPLOYMENT
──────────
Developer → git push → GitHub → Cloudflare Pages (auto-deploy)

DNS CHAIN
─────────
Porkbun (registrar)
  └─→ Cloudflare DNS (nameservers)
        ├─→ adsvizor.com          → Cloudflare Pages
        ├─→ www.adsvizor.com      → Cloudflare Pages
        ├─→ *.adsvizor.com        → Cloudflare Workers (per client)
        └─→ contact@adsvizor.com  → Gmail (Porkbun email routing)

DAILY AI LOOP (planned)
────────────────────────
Scheduler → Google Sheets (READ leads)
          → Google Ads API (READ metrics)
          → Claude API (PROCESS → ranked recommendations JSON)
          → Google Ads API (OPTIONAL: apply changes)
          → Storage (log output with run_id)

9. Security & Credentials

  • No API keys committed to the repository (.cursorrules enforces this)
  • Google Apps Script endpoint uses a deployment ID token in the URL (publicly visible but scoped)
  • Cloudflare Pages/Workers authentication is managed at the account level
  • Google Sheets access is granted via Apps Script built-in permissions — no explicit keys needed
  • Porkbun/DNS credentials are managed at the registrar level only
